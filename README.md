# Javascript without IA

An in-depth repository dedicated to mastering pure JavaScript by relying solely on official documentation, MDN, and core programming principles—completely free of AI assistance.
The core purpose of this project is to build strong muscle memory, master problem-solving from scratch, and enforce industry best practices.

### Phase 0: Core Fundamentals


| What                                         | Rule                                                                                                     |  Why                        |
| :--------                                    | :-------------------------                                                                               |  :------------------------- |
| 0. Atomic & Semantic Commits                 | Every commit must follow a strict naming convention                                                      | Ensures a clean, readable, and professional Git history that team members (and future employers) can easily track.|
| 1. Meaningful Naming Conventions             | Variables, functions, and classes must have clear, descriptive names that explicitly state their intent. | Code should be self-documenting. Avoid cryptic abbreviations or generic names like data, temp, or x.|
| 2. The Single Responsibility Principle (SRP) | Functions must be small, simple, and isolated. A function should do one thing and do it well.            | Complex, "do-it-all" functions are prone to bugs, difficult to test, and hard to maintain. |
